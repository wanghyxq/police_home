# 用户角色动态展示计划

根据需求，我们将修改 `pages/mine/index.vue` 中硬编码的“民警”标签，改为根据用户角色动态显示。

## 1. 业务逻辑

*   **角色获取**：从 Vuex store (`$store.state.user.roles`) 获取当前用户的角色列表。
*   **角色映射**：
    *   `admin`: 显示 "管理员"
    *   `barber`: 显示 "理发师"
    *   `common` 或其他: 显示 "民警" (默认)
*   **优先级**：如果用户有多个角色，优先显示特定角色（如理发师、管理员）。

## 2. 代码实现

1.  **修改 `computed` 属性**：
    *   在 `pages/mine/index.vue` 中新增 `roleName` 计算属性。
    *   逻辑：
        ```javascript
        roleName() {
          const roles = this.$store.state.user.roles || [];
          if (roles.includes('admin')) return '管理员';
          if (roles.includes('barber')) return '理发师';
          return '民警';
        }
        ```
2.  **更新模板**：
    *   将硬编码的 `民警` 替换为 `{{ roleName }}`。

## 3. 实施步骤

1.  **修改 `pages/mine/index.vue`**：添加 `roleName` 计算属性并更新模板插值。
2.  **验证**：由于当前是民警端个人中心，通常只会显示“民警”或“管理员”。理发师有独立的个人中心页面，但也应确保此处逻辑正确以防万一。
