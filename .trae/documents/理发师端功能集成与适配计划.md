# 理发师端功能集成计划

基于提供的原型文件 `理发师端01222322.html`，我们将把理发师端功能集成到现有的 uni-app 项目中。

## 1. 页面拆分与重构

我们将原型中的单页应用逻辑拆分为以下独立的 Vue 页面组件，放置在 `pages/barber/` 目录下：

*   **`pages/barber/dashboard.vue`**: 理发师工作台（对应原型的 `page-barber-dashboard`）。包含今日数据统计和近期预约列表。
*   **`pages/barber/history.vue`**: 历史预约（对应原型的 `page-barber-history`）。包含按月/周筛选的历史记录列表。
*   **`pages/barber/profile.vue`**: 个人中心（对应原型的 `page-barber-me`）。复用并适配 `pages/mine/index.vue` 的样式风格。
*   **`pages/barber/detail.vue`**: 预约详情（对应原型的 `page-barber-detail`）。包含订单信息、操作按钮（确认完成）和评价展示。

**注意**：原型中的登录页 `page-login` 不再需要，项目已存在通用的 `pages/login.vue`，我们将直接复用现有登录逻辑，通过角色判断跳转到不同的首页。

## 2. 路由配置 (`pages.json`)

我们将更新 `pages.json`，注册上述新页面。

```json
{
  "pages": [
    // ... 原有页面
    {
      "path": "pages/barber/dashboard",
      "style": { "navigationBarTitleText": "理发工作台" }
    },
    {
      "path": "pages/barber/history",
      "style": { "navigationBarTitleText": "历史预约" }
    },
    {
      "path": "pages/barber/profile",
      "style": { "navigationBarTitleText": "个人中心" }
    },
    {
      "path": "pages/barber/detail",
      "style": { "navigationBarTitleText": "预约详情" }
    }
  ]
}
```

## 3. 样式与 UI 还原

我们将使用 Tailwind CSS 类名（项目中已通过 unocss/windicss 支持）来还原原型样式，并复用项目现有的 `haircut.css`。

*   **配色**：保持原型的警蓝 (`text-police`, `bg-police`) 和灰色调。
*   **组件**：
    *   **Tabbar**：由于 uni-app 的原生 Tabbar 不支持根据角色动态切换，我们将为理发师端封装一个独立的底部导航组件 `pages/barber/components/barber-tabbar.vue`，在 `dashboard`、`history`、`profile` 页面引入。
    *   **列表项**：还原原型中的卡片式布局。

## 4. 现有页面优化 (`pages/mine/index.vue`)

根据要求，我们将优化现有的“我的”页面，使其视觉风格与首页门户保持一致：
*   顶部背景改为蓝色 (`bg-police`)。
*   调整头像和用户名的布局与颜色（白字）。
*   优化菜单列表的样式。

## 5. 实施步骤

1.  **创建组件**：新建 `pages/barber/components/barber-tabbar.vue`。
2.  **创建页面**：依次创建 `dashboard.vue`, `history.vue`, `profile.vue`, `detail.vue`，并填充布局和模拟数据逻辑（后续对接真实接口）。
3.  **配置路由**：更新 `pages.json`。
4.  **优化旧页面**：修改 `pages/mine/index.vue`。
5.  **验证**：检查页面跳转、参数传递和样式表现。

此计划将确保理发师端功能无缝融入现有项目，同时保持代码结构清晰。
